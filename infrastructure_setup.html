Infrastructure Setup¶

The infrastructue setup before setting up the actual Cloud Builder service instances consists out of two steps:

  1. Create a git repository to store the package sources

  2. Create a message broker service for the communication between Cloud Builder services

The following picture shows how the Cloud Builder services play together and provides a visual overview about the system.

Cloud Builder Architecture¶

Note

Don’t be scared about the complexity of the diagram above. The architecture as shown there shows a complete landscape of a build backend as it could be used to build a Linux distribution. The minimal infrastructure to build packages with Cloud Builder requires:

  • A git repo with the package sources. There is the cloud-builder-packages example repo

  • A Kafka message broker. The focus will be on Amzon MSK

  • A runner instance.

As the idea of Cloud Builder is to come up with a scalable system that is configurable to the needs of the user, most of the services are optional on top of the minimal infrastructure.
